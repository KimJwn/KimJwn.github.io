# 호주 딥페이크 관련 법 요약

## 핵심 법령/제도
- `Criminal Code Amendment (Deepfake Sexual Material) Act 2024 (C2024A00078)`: 비동의 성적 딥페이크 영역의 형사 규율을 직접 강화한 핵심 법률 축. (Ref1)
- `Online Safety ... Industry Standard 2024 (F2024L00710)`: 온라인 서비스 사업자의 운영·준수 의무를 구체화하는 집행형 규범. (Ref2)
- `Criminal Code Act 1995 Compilation No. 171 (Includes amendments: Act No. 2, 2026)`: 통합본 관점에서 법체계 연결과 적용 맥락을 확인할 수 있는 자료. (Ref4, Ref5, Ref6)

## 요약
- 호주는 딥페이크 전면 단일법 체계보다는, 특정 위해(특히 비동의 성적 합성물)에 대한 형사 조문 강화와 온라인 안전 규범(산업표준) 기반 플랫폼 의무를 결합하는 구조입니다. (Ref1, Ref2)
- 즉 “형사 처벌 강화 축 + 서비스 제공자 운영의무 축”이 병렬로 작동하며, 실제 리스크 관리는 두 축을 함께 설계할 때 가장 효과적입니다. (Ref1, Ref2, Ref4)

## Policy Dimensions
- `라벨링(표시) 법제화 상태`: 현 리소스 기준, 모든 맥락을 단일 조문으로 포괄하는 일반 딥페이크 라벨링 의무가 전면적으로 분리되어 있다고 보기는 어렵습니다. 다만 온라인 안전·산업표준 규범에서 서비스 제공자의 공지·표시·운영 통제와 결합되는 방식으로 실무 라벨링이 작동하는 구조가 확인됩니다. 따라서 라벨링 준수는 법률 문구 자체보다 플랫폼 운영정책 설계와 결합해 해석해야 합니다. (Ref2, Ref3)
- `선거 관련 법제화 상태`: 현재 공식 로컬 코퍼스만으로는 선거 딥페이크만을 독립적으로 포괄하는 연방 단일법의 명시 근거가 제한적입니다. 따라서 선거 영역은 일반 온라인 안전 체계 및 기존 선거/허위정보 규율과의 결합 집행 가능성을 전제로 보수적으로 기술하는 것이 안전합니다. (Ref2)
- `동의 없는 딥페이크 법제화 상태`: 비동의 성적 딥페이크는 호주에서 가장 직접적으로 법제화된 축입니다. `Criminal Code Amendment (Deepfake Sexual Material) Act 2024`는 해당 위해 유형에 대해 형사 집행 경로를 명확히 강화하며, 서비스 운영 측면에서도 피해 확산 억제를 위한 신속 대응 설계가 요구됩니다. (Ref1, Ref4)
- `서비스 제공자 및 플랫폼 책임 법제화 상태`: 플랫폼 책임은 온라인 안전 체계와 산업표준에서 비교적 구체적으로 운영됩니다. 핵심은 단순 삭제 의무를 넘어서 접수·분류·조치·기록·보고 흐름을 규정 준수 가능한 형태로 유지하는 것이며, 기술적 통제와 규제기관 대응 체계를 함께 갖춰야 합니다. (Ref2, Ref3)
- `처벌 법제화 상태`: 처벌은 특히 비동의 성적 딥페이크 영역에서 형사 조문 강화가 뚜렷하고, 통합본(Compilation) 문서 맥락에서 관련 조문 체계와 적용 연결을 추적할 수 있습니다. 실무적으로는 최초 위반뿐 아니라 반복/미이행 상황까지 포함한 제재 노출 관리를 해야 합니다. (Ref1, Ref4, Ref5, Ref6)

## 실무 포인트 (Dimension별)
- `라벨링(표시)`: 라벨링 문구, 노출 위치, 사용자 인지 가능성(가시성/지속성)을 제품 릴리스 체크리스트에 고정 항목으로 넣고 증빙 로그를 남겨야 합니다.
- `선거`: 선거기간 관련 콘텐츠는 일반 모더레이션과 분리된 별도 심사 게이트를 두고, 오탐/이의제기 처리 시간을 사전에 정의해야 합니다.
- `동의 없는 딥페이크`: 피해신고 접수 즉시 차단, 재업로드 탐지, 증거보존(원본 URL·해시·타임스탬프)까지 이어지는 SOP를 표준화해야 합니다.
- `서비스 제공자/플랫폼 책임`: 규정 문서(정책)와 실제 조치 로그(운영)를 1:1로 연결해 감사/규제질의 시 즉시 추적 가능하도록 운영해야 합니다.
- `처벌`: 1차 위반, 시정요구 이후 불이행, 반복위반의 단계별 제재 시나리오를 리스크 매트릭스로 관리해야 합니다.

## 확인 필요
